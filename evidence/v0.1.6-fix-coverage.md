## Implication for the L2 R2 run

An R2 with v0.1.6 genuinely re-tests **only #34** (over-read on redis, tokio,
typescript, hugo, bitcoin). For the #32 repos (typescript, webpack) and #31 repos
(redis, django, webpack, spring-boot, rails, laravel), v0.1.6 still has the bug,
so R2 will **reproduce** the friction, not measure a fix's bite.

To R2 all three issues properly, build a grove binary that also merges #31 + #32
+ #33 (none of which are in v0.1.6), then rebuild the `grove` image from that.
